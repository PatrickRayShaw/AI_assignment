# XU-News-AI-RAG 概要设计文档

## 1. 系统概述

个性化新闻智能知识库系统，采用 RAG（检索增强生成）架构，实现从新闻采集到智能问答的全链路自动化。

## 2. 技术架构

```
┌─────────────────────────────────────────────────┐
│                    前端层                        │
│  Vue 3 + Element Plus + Vite                    │
│  Login / Dashboard / Knowledge / Search /       │
│  Cluster / Settings                             │
└──────────────────┬──────────────────────────────┘
                   │ HTTP/REST (JWT Auth)
┌──────────────────▼──────────────────────────────┐
│                   后端层                         │
│  Flask + Flask-CORS + PyJWT                     │
│  Routes: auth / knowledge / search / cluster    │
│  Services: search_service / cluster_service     │
└──────┬──────────────────────┬───────────────────┘
       │                      │
┌──────▼──────┐    ┌──────────▼──────────┐
│   SQLite    │    │  FAISS 向量数据库    │
│  用户/新闻   │    │  语义检索 / 聚类     │
│  标签/日志   │    │  Embedding 存储     │
└─────────────┘    └─────────────────────┘
       │                      │
┌──────▼──────────────────────▼──────────┐
│              AI 模型层                  │
│  Ollama: qwen2.5:3b (本地推理)         │
│  Embedding / 问答生成 / 摘要           │
└────────────────────────────────────────┘

┌────────────────────────────────────────┐
│          工作流引擎层 (n8n)             │
│  RSS采集 → 清洗 → 入库 → 向量化        │
│  Webhook → Embedding → FAISS → 返回    │
│  定时任务 / 事件触发 / 邮件通知        │
└────────────────────────────────────────┘
```

## 4. n8n 工作流设计

### 4.1 新闻采集工作流 (news_collection.json)
```
RSS源 → HTTP Request → HTML解析 → 数据清洗
  → 重复检查 → SQLite写入 → FAISS向量化
  → 邮件通知(新新闻摘要)
触发: 定时 (每小时)
```

### 4.2 语义检索工作流 (semantic_search.json)
```
Webhook接收查询 → Embedding(查询文本)
  → FAISS相似度检索 → 结果排序 → JSON返回
触发: HTTP Webhook
```

### 4.3 联网兜底工作流 (web_fallback.json)
```
检索置信度检查 → 低于阈值 → 联网搜索
  → 结果格式化 → 合并返回
触发: 语义检索子流程
```

### 4.4 聚类分析工作流 (cluster_analysis.json)
```
FAISS向量聚类 → 簇标签生成
  → 每簇摘要提取 → JSON结果
触发: 手动/API
```

### 4.5 邮件通知工作流 (email_notify.json)
```
新新闻入库事件 → 模板渲染 → SMTP发送
触发: 新闻采集完成后
```

## 5. API 接口设计

### 认证模块 (auth)
- `POST /api/auth/register` — 用户注册
- `POST /api/auth/login` — 用户登录，返回JWT

### 知识库模块 (knowledge)
- `GET /api/knowledge/list` — 分页查询新闻列表
- `GET /api/knowledge/<id>` — 新闻详情
- `DELETE /api/knowledge/<id>` — 删除新闻

### 检索模块 (search)
- `POST /api/search/semantic` — 语义检索
- `POST /api/search/web` — 联网搜索
- `GET /api/search/history` — 搜索历史

### 聚类模块 (cluster)
- `POST /api/cluster/analyze` — 执行聚类分析
- `GET /api/cluster/results` — 获取聚类结果

## 7. 安全设计

- JWT 令牌认证，过期时间 24 小时
- 密码 bcrypt 哈希存储
- SQL 参数化查询防注入
- CORS 白名单限制
- n8n Webhook 路径令牌验证
