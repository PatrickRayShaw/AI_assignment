# XU-News-AI-RAG：个性化新闻智能知识库 — 产品需求文档（PRD）

## 1. 引言

### 1.1 背景
在信息爆炸的时代，用户每天面对海量新闻资讯，难以高效筛选和检索有价值的信息。传统新闻聚合工具缺乏智能语义理解和个性化知识管理能力。本项目旨在构建一个基于 RAG（检索增强生成）技术的个性化新闻智能知识库系统，实现新闻自动采集、智能检索、大模型问答与知识聚类分析。

### 1.2 目标用户
- 信息工作者、研究人员：需要高效检索和整理新闻资讯
- 内容创作者：需要追踪特定领域的最新动态
- 普通用户：希望获得个性化、去噪的新闻摘要

### 1.3 产品愿景
打造一个"采集→入库→检索→分析→通知"全链路自动化的私有化新闻知识库，用户只需登录即可获得智能化的新闻管理与问答体验。

## 4. AI产品特定需求

### 4.1 模型需求
| 模型 | 用途 | 性能指标 |
|------|------|----------|
| qwen2.5:3b (Ollama) | 问答推理、摘要生成 | 响应<5s，准确率>85% |
| all-MiniLM-L6-v2 | 文本嵌入向量化 | 768维，余弦相似度>0.7 |
| ms-marco-MiniLM-L-6-v2 | 检索结果重排序 | MRR@10 > 0.35 |

### 4.2 数据需求
- 来源：RSS订阅源、网页抓取（合规）
- 数据类型：结构化（Excel可导入）+ 非结构化（文本/HTML）
- 存储：元数据→MySQL/SQLite；向量→FAISS
- 更新频率：n8n定时触发（可配置，默认每小时）

### 4.3 算法边界与可解释性
- 向量检索：返回Top-K结果附带相似度分数
- 大模型回答：标注"基于知识库"或"基于联网搜索"
- 聚类：TF-IDF + K-Means，输出关键词权重

### 4.4 评估标准
- 检索准确率：Top-5召回率 > 80%
- 系统可用性：99.5% uptime
- 响应延迟：搜索<2s，问答<5s

### 4.5 伦理与合规
- 新闻采集严格遵守robots.txt
- 所有数据本地处理，不上传第三方云
- 用户数据加密存储

## 6. n8n工作流专项需求

### 6.1 工作流场景设计
1. **新闻采集工作流**：定时触发→RSS读取→网页抓取→数据清洗→入库
2. **语义检索工作流**：接收查询→向量检索→相似度排序→结果返回
3. **联网兜底工作流**：判断匹配度→百度搜索API→大模型推理→输出
4. **聚类分析工作流**：数据统计→TF-IDF→K-Means→生成报告
5. **邮件通知工作流**：入库触发→邮件模板→发送通知

### 6.2 节点选型
| 节点类型 | 用途 |
|----------|------|
| Schedule Trigger | 定时触发采集 |
| RSS Read | 读取RSS源 |
| HTTP Request | 网页抓取、Ollama API调用、百度搜索API |
| Function | 数据清洗转换 |
| Database | MySQL/SQLite读写 |
| FAISS (Custom) | 向量存储与检索 |
| IF | 相似度阈值判断 |
| Email | 邮件发送 |
| JWT Auth | 身份认证 |

### 6.3 触发条件与异常处理
- 定时触发：每小时执行，失败重试3次
- 入库触发：成功后触发邮件通知
- 异常处理：错误日志记录、降级策略

### 6.4 私有化部署合规
- n8n本地Docker部署
- 数据库本地存储
- 无第三方云依赖（除百度搜索API可选）
